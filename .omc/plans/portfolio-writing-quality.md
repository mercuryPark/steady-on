# 포트폴리오 글 품질 개선 계획

> AI가 쓴 느낌 제거 + 중복 제거 + 가독성 개선

---

## 원칙

1. **사람이 쓴 글** — "~하고, ~하여, ~를 통해" 접속 나열 금지. 짧은 단문 기본. 한 항목 1~2문장.
2. **중복 금지** — 같은 내용은 가장 적합한 곳 1군데에만. 나머지는 삭제 또는 다른 관점으로.
3. **성질 맞추기** — About Me는 성향/가치관, Career는 업무 요약, 타임라인은 구체 작업, Performance는 수치 성과.
4. **문장 구조** — `[짧은 핵심] + [구체적 수치/맥락]` 패턴. 형용사 대신 수치.

## 수정 계획 (4 Step)

### Step 1: Contents.tsx — About Me 롤백 + 자연스러운 AI 항목

**수정 범위**: `Contents.tsx:106-111`

**변경 전** (현재):
```
"Claude Code 기반 바이브코딩 워크플로우를 직접 설계하고, CLAUDE.md 컨텍스트 관리와 커스텀 커맨드로 개발 프로세스를 자동화해요."
"Biome → Vitest → Playwright 4단계 테스트 파이프라인을 구성해 CI에 연동하고, AI 에이전트 코딩에 최적화된 리뷰 프로세스를 운영해요."
```

**변경 후**:
```
"새로운 도구와 방법론을 실무에 빠르게 적용해보는 걸 좋아해요."
```

**이유**: About Me는 "나는 이런 사람" — 기술 스펙 2줄을 성향 1줄로 교체. Claude Code, 테스트 파이프라인 상세는 Performance에서 충분히 다룸.

---

### Step 2: Career.tsx — AI 업무 항목 간결화 + 중복 제거

**수정 범위**: `Career.tsx:84-85`

**변경 전** (현재):
```
"Claude Code 바이브코딩 워크플로우 설계 및 신규 기능·마이그레이션 3개 프로젝트에 전면 적용 (일정 75% 단축)"
"Biome → Vitest → Playwright 4단계 테스트 자동화 파이프라인 구축 및 CI 연동"
```

**변경 후**: 2줄 → 1줄로 통합
```
"Claude Code 활용 개발 워크플로우와 테스트 자동화 환경 구축"
```

**이유**: Career는 업무 요약 목록. 수치(75%)와 도구 상세(Biome → Vitest → Playwright)는 Performance 카드에서 다루므로 여기서는 한 줄 요약으로 충분. "바이브코딩", "전면 적용", "파이프라인" 등 키워드 적재 제거.

---

### Step 3: Cureer.tsx — 타임라인 문장 다듬기

**수정 범위**: 2곳

**3-1. 메신저 마이그레이션 (Cureer.tsx:188)**

변경 전:
```
"기존 메신저 인증 시스템 및 개발 환경의 Next.js 프레임워크 전환 마이그레이션을 Claude Code 바이브코딩 워크플로우로 주도"
```

변경 후:
```
"기존 인증 시스템과 개발 환경을 Next.js로 전환. Claude Code를 활용해 마이그레이션을 주도"
```

**이유**: "바이브코딩 워크플로우"라는 수식을 빼고, "Claude Code를 활용해"로 간결하게. 타임라인 항목은 사실 나열이 자연스러움.

**3-2. Admin 마이그레이션 (Cureer.tsx:222)**

변경 전:
```
"Claude Code 워크플로우를 적용해 어드민 개발 환경의 Next.js 전환(Nuxt.js 기반 레거시 탈피) 및 주요 정책/로그/현황 페이지 마이그레이션"
```

변경 후:
```
"Nuxt.js → Next.js 전환 및 주요 페이지 마이그레이션. Claude Code로 전환 작업 가속"
```

**이유**: 괄호 설명 제거, "레거시 탈피" 같은 과장 표현 제거. 짧은 2문장으로 분리.

---

### Step 4: Performance.tsx — AI 카드 문장 다듬기

**수정 범위**: `Performance.tsx:59, 63`

**4-1. 첫 번째 항목 (line 59)**

변경 전:
```
"CLAUDE.md 기반 프로젝트 컨텍스트 관리와 커스텀 커맨드(git-commit, test-legacy, test-new-feature)로 개발 프로세스를 자동화하고, MCP 서버 연동으로 마이그레이션 시 deprecated API 없이 전환 → 기존 일정 대비 75% 단축"
```

변경 후:
```
"CLAUDE.md로 프로젝트 규칙을 정의하고, 커스텀 커맨드로 커밋·테스트를 자동화. MCP 서버를 연동해 마이그레이션 시 최신 API만 사용하도록 세팅. 기존 일정 대비 75% 단축."
```

**이유**: 한 문장을 3문장으로 분리. 각각 독립적으로 읽힘. "컨텍스트 관리", "개발 프로세스를 자동화하고" 같은 AI 특유의 추상 표현 제거.

**4-2. 두 번째 항목 (line 63)**

변경 전:
```
"프로젝트별 wiki, docs, coverage 등 진행 상황 자동 문서화 프로세스를 구축하고, AI가 생성한 계획·실행 기록을 리뷰하는 에이전트 코딩 최적화 리뷰 방식을 도입해 운영 중"
```

변경 후:
```
"프로젝트별 wiki와 커버리지 리포트를 자동 생성하도록 구성. 코드 한 줄 단위가 아닌, AI가 만든 계획과 실행 기록을 리뷰하는 방식으로 전환해 운영 중."
```

**이유**: "진행 상황 자동 문서화 프로세스를 구축하고" → "자동 생성하도록 구성". "에이전트 코딩 최적화 리뷰 방식을 도입해" → "~방식으로 전환해". 자연스러운 서술.

## 검증 기준

- [ ] "바이브코딩"이라는 단어가 포트폴리오 전체에서 2회 이하로 등장 (Performance 카드 제목 + description만)
- [ ] "Biome → Vitest → Playwright" 전체 나열이 1곳(Performance 카드)에서만 등장
- [ ] "75% 단축"이 1곳(Performance 카드)에서만 등장
- [ ] About Me 항목이 모두 성향/가치관/습관 성격의 문장
- [ ] Career tasks에 3단 이상 접속 문장("~하고, ~하여, ~를 통해") 없음
- [ ] 타임라인 항목이 각각 1~2문장 이내
- [ ] Performance 카드 각 항목의 문장이 독립적으로 읽힘 (3단 접속 없음)
- [ ] `npm run build` 성공
