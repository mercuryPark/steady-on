# git-commit skill

한국어 커밋 메시지를 프로젝트 컨벤션에 맞게 자동 생성하고 커밋합니다.

## 실행 절차

1. `git status`로 변경 파일 확인
2. `git diff --staged`로 스테이징된 변경 사항 분석 (스테이징된 것이 없으면 `git diff`로 전체 변경 확인)
3. `git log --oneline -10`으로 최근 커밋 스타일 참조
4. 변경 내용에 맞는 카테고리 자동 선택
5. 한국어로 간결한 커밋 메시지 생성
6. 사용자에게 메시지 확인 후 커밋 실행

## 커밋 메시지 포맷

```
[카테고리] 설명
```

### 카테고리 규칙

| 카테고리 | 사용 시점 | 예시 |
|----------|----------|------|
| `기능` | 새로운 기능 추가 | `[기능] 블로그 검색 기능 추가` |
| `수정` | 버그 수정 | `[수정] 모바일에서 카드 레이아웃 깨지는 문제 해결` |
| `스타일` | UI/CSS 변경 (기능 변화 없음) | `[스타일] 포스트 카드 호버 애니메이션 개선` |
| `리팩터` | 코드 구조 개선 (기능 변화 없음) | `[리팩터] 프로필 컴포넌트 타입 정리` |
| `문서` | 문서 수정 | `[문서] CAREER.md 경력 사항 업데이트` |
| `설정` | 설정/빌드/의존성 변경 | `[설정] Tailwind 커스텀 색상 추가` |
| `콘텐츠` | 블로그 포스트 추가/수정 | `[콘텐츠] React 성능 최적화 포스트 작성` |
| `리뉴얼` | 대규모 개편/리디자인 | `[리뉴얼] 포트폴리오 내용 및 디자인 보완` |

### 카테고리 판단 기준

- `content/blog/` 변경 → `콘텐츠`
- `docs/` 변경 → `문서`
- `gatsby-config.js`, `package.json`, `tailwind.config.js` 등 설정 파일만 변경 → `설정`
- `.css`, `.style` 관련만 변경 → `스타일`
- 여러 영역에 걸친 대규모 변경 → `리뉴얼`
- 컴포넌트 내부 로직 변경 없이 구조만 변경 → `리팩터`
- 그 외 새 기능 → `기능`, 버그 → `수정`

## 메시지 작성 규칙

- 한국어로 작성
- 50자 이내로 간결하게
- "~추가", "~수정", "~개선", "~변경", "~보완" 등 동사로 끝내기
- 무엇을 했는지 명확하게 (파일명이 아닌 기능/내용 중심)
- 여러 변경이 있으면 가장 핵심적인 변경을 설명

## 스테이징 규칙

- 변경 파일이 스테이징되어 있지 않으면, 관련 파일을 개별적으로 `git add` (git add -A 사용 금지)
- `.env`, 시크릿 파일은 절대 스테이징하지 않음
- `public/`, `.cache/`, `node_modules/` 는 .gitignore 대상이므로 무시

## 커밋 실행

```bash
git commit -m "$(cat <<'EOF'
[카테고리] 설명

Co-Authored-By: Claude <noreply@anthropic.com>
EOF
)"
```
