## Key Strengths

**프론트엔드 설계 & 디자인 시스템**
B2B 웹 서비스와 어드민 툴을 초기 설계부터 운영까지 담당. Toast, FAB, 검색 필터, 버튼 등 공용 컴포넌트를 variants·스타일 props 기반으로 설계해 여러 서비스에서 공통 사용. 디자인 토큰(컬러·글꼴·테마)도 함께 관리.

**성능 최적화**
IndexedDB 캐싱으로 반복 API 호출을 없애고, Web Worker로 UI 프리징을 해결. LCP를 6초에서 3초대로 줄인 경험. Vite 번들 분석과 코드 스플리팅도 직접 수행.

**상태관리에 대한 깊은 이해**
Jotai의 atom 모델을 분석해서 의존성 기반 구독 최소화를 직접 구현한 경량 라이브러리(piko)를 만들었고, 실무에서는 React Query로 서버 상태를 관리.

**AI 기반 개발 자동화 & 바이브코딩**
Claude Code 기반 바이브코딩 워크플로우를 직접 설계하고 실무에 적용. CLAUDE.md로 프로젝트 컨텍스트를 관리하고, 커스텀 커맨드(git-commit, test-legacy, test-new-feature)와 MCP 서버를 연동해 개발 프로세스를 자동화. Biome → Vitest → Playwright 4단계 테스트 파이프라인을 구성해 CI에 연동하고, 신규 기능 3건과 프레임워크 마이그레이션 3건에 적용해 일정 75% 단축 후 배포 완료.

### AI 기반 개발 워크플로우 설계 & 바이브코딩 실무 적용

**바이브코딩 워크플로우 설계**

- CLAUDE.md에 프로젝트별 훅과 규칙을 정의해서, 프롬프트만으로도 컨벤션에 맞는 코드가 나오도록 세팅
- git-commit, test-legacy, test-new-feature 같은 커스텀 슬래시 커맨드를 만들어서 커밋 메시지 작성 시간을 3~5분에서 1분 이내로 줄임
- MCP 서버로 최신 라이브러리 문서를 참조하게 해서 마이그레이션할 때 deprecated API 없이 깔끔하게 전환

**테스트 자동화 파이프라인**

- Biome(정적 분석) → Vitest(단위) → 통합 테스트 → Playwright(E2E)로 이어지는 4단계 테스트 파이프라인을 설계하고 CI에 연동. 새 기능을 넣을 때 기존 기능이 깨지는지 자동으로 확인되는 구조

**AI 에이전트 코딩에 최적화된 리뷰 프로세스**

- 프로젝트별 wiki, docs, coverage 등 진행 상황 요약과 히스토리를 자동 생성하는 프로세스를 구축
- 엄격한 라인 단위 코드리뷰 대신, AI가 생성한 계획·실행 기록·변경 이력을 리뷰하는 에이전트 코딩 최적화 리뷰 방식을 도입해 운영 중

**실무 적용 & 성과**

- 메신저 신규 기능 3건에 이 워크플로우를 전면 적용해서 기존 예상 일정의 1/4 수준으로 개발을 끝내고 배포 완료
- 메인 서비스 포함 총 3개 프로젝트의 프레임워크 마이그레이션을 주도하고 완료
- 토큰 관리, 작업 계획, Claude 세팅을 계속 고도화하면서 현재도 실무에서 운영 중
- 개발 과정의 의사결정과 변경 사항은 .md로 문서화해서 히스토리를 추적할 수 있게 관리
